1、混淆矩阵

混淆矩阵（Confusion Matrix），又称为可能性矩阵或错误矩阵，在监督学习中一般叫做匹配矩阵，在图像精度评价中或分类任务中主要用于比较分类结果和实际测得值，可以直观显示出分类结果的精度

预测值作为列标，实际值作为行标，单元格内容表示：满足预测值为该单元格所在列对应的类别，且实际值为该单元格所在行对应的类别的样本数（或比例）

因此，一个一般的混淆矩阵会长成下面这种样子：

(实际值分类)\(预测值分类) 类1 类2 类3
类1 43 2 0
类2 5 45 1
类3 2 3 49

接下来列出能够从混淆矩阵中得出来的一些分类指标（以二分类问题示例）：

（假设结果是属于某模型的）

\n

设：$TP$ 为左上角值，$FN$ 为右上角值，$FP$ 为左下角值，$TN$ 为右下角值

则：

精确率$=\frac{TP+TN}{TP+FN+FP+TN}$

正确率$=$准确率$=\frac{TP}{TP+FP}$

召回率$=$查全率$=$敏感指标（True Positive Rate, TPR）$=\frac{TP}{TP+FN}$

由此可以继续定义出 $F_\beta_Score$ 用以以不同权重配比（$\beta$）衡量模型的精确度

精确率和精确度是判断分类模型总体的标准（区别于计算某一分类的特性的标准）

2、元胞自动机

实际上，之前刷到的“模拟生命演化”之类的，用的就是元胞自动机

元胞自动机把连续的空间划分为一个又一个细胞，即离散化，并且将时间也一并离散化（实际上这是计算机模拟连续过程所必须做的一步（应该吧）），每一时刻根据一定的规则，并行地计算每一个细胞周围的细胞对其造成的影响，据此并行地更新每一个细胞的状态，实现对实际空间演化情况的模拟

“周围”有若干种，如：

显然，“周围”的范围越大，“一定的规则”就会越复杂

同时，由于元胞自动机是一种自动机，而自动机就有状态转移方程，因此每个细胞可取的状态数越多，“一定的规则”也会越复杂，“一定的规则”就是状态转移方程（而不是别的什么，只是因为自己复述概念的时候这样说，所以也给它加上了双引号）

当元胞自动机的状态转移方程变得非常复杂的时候，其内部所需要确定的参数也会变得越来越多

如果需要制作出一个数学模型，使得元胞自动机能够通过模拟给出对未来空间状态的一份预测，那么确定参数将成为一个重要的问题

如果掌握过往的实际测得值，就可以尝试使用算法（近似地）求解这些参数

这个时候，就有了将如启发式算法、神经网络等其他算法与元胞自动机结合起来的应用方法

因此此时，其他算法的应用目标在于确定状态转移方程的参数

3、龙格-库塔方法（Runge-Kutta）

参考文章（CSDN）

用以求常微分方程的数值解（在一定精度下）

这种方法能够达到的精度由数学分析给出（主要是泰勒展开，贯穿了整个方法）

首先假定一个前提条件：出现的所有函数都连续可导

那么来说首先泰勒展开就是对的

按照由 $x_n$（递推）计算 $x_{n+1}$ 的思路进行分析，发现每次计算的关键在于算出区间 $(x_n, x_{n+1})$ 上的平均斜率 $k$

于是考虑怎么把这个 $k$ 算出来，龙格-库塔方法考虑通过取点算斜率的加权平均作为 $k$，并且对此妥善构造以达到更高精度，这就是此方法的基本思想

规律：$n$ 阶龙格-库塔方法能够达到 $n$ 阶精度（即余项 $o(h^n)$）（补充：“参考文章”里写的是 $O(h^{n+1})$）

优点：

  1. 单步法，我的理解是能够把各点的数值一个一个递推出来
  2. 数值稳定性较好（==不知道什么意思==）
  3. 能够自启动，只要给出一个点就能开始往后算，一般微分方程要求给出初始值，因此此方法一般能够运行
  4. 编程实现容易

在区间 $[x_n, x_{n+1}]$ 内取 $n$ 个点（包含 $x_n$）并且满足一定条件（见“参考文章”）可以达到 $n$ 阶精度

这个 $h$ 是步长，$\Delta x$，求数值解一般是求出常微分方程解函数在 $x_i, i=1,2,3\dots$ 处的函数值（在初始值约束下），那么这个 $h=\Delta x$ 就是 $|x_n-x_{n+1}|$（一般步长固定不变，==也有步长可变的求解方法==）

4、置信度

DeepSeek 讲授

重要的新认知点：

置信度是对置信区间而言的，表示“用同样的抽样方法，对某个目标参数值重复多次计算得出多个置信区间，那么这一系列置信区间中，大约有多少置信区间会包含总体的目标参数值（真实值）”

抽样的目的就是要通过实际观测少量样本的目标参数值，据此估计出总体的目标参数值

需要抽样的原因还是经典原因：总体规模太大，没办法一一观测

抽样造成计算得出的样本的目标参数值注定与总体的目标参数值之间存在误差

因此，一般根据样本的观测值，计算得出一个置信区间，并且控制置信度，以使抽样结果有实际意义（“能用”）

置信区间的计算公式中，包含：

  1. 样本统计量（抽样观测直接得到）
  2. 临界值（取决于样本量与总体服从的分布）
  3. 标准误（取决于样本标准差与样本量，此二者由抽样观测直接得到）

置信度越高，置信区间越宽，反之亦然

（实际上 25春季学期 物理实验后的数据处理就用到了这个知识点，只是那个时候只会套公式去了）

5、数学规划模型初步

系 《数学模型》第四版，姜启源等编 读书笔记切片

敏感度分析：模型的系数对结果的取值特征的影响

影子价格：在当前取值下个变量增加（减少）一单位会对目标函数造成多少的增益（亏损）

但实际上它的副作用是有限制的呈现为某变量最多增（减）多少单位

这一指标可以为是否继续投资提供参考

剩余资源：在最优解下各变量取值代入后与个约束右端值之间的差距，若差距为零则该约束为紧约束

LINGO 用于解优化模型（所以面对数学规划问题，如线性规划问题，就可以用这个软件求解）

分析结果中，有一项指标叫做 Reduced Cost，有结论：
$$
Reduced\ Cost>0 \Rightarrow x=0.
$$
对这个指标的其中一种理解是：如果这个变量多一单位，那么目标函数值会少多少

6、代数方程与差分方程模型初步

系 《数学模型》第四版，姜启源等编 读书笔记切片

量纲齐次原则

分析量纲，获得一个代数方程，用矩阵表示，有 $\pi$ 定理指导列出最终的变量之间的关系式

蛛网模型

这一时期的产品数量决定本时期的价格，本时期的价格决定下一时期的产品数量，循环往复，在两个函数之间来回跳转取点，将上一个点与下一个点用直线连起来，织成一个“蛛网”，即蛛网模型

蛛网有两种基本结局：收敛与发散

一般来说，两个函数的交点即为平衡点，当取得该点之后，每一步都将取得该点，无法逃离，过程达成稳定

（当然实际生活中这是不可能的）

一般来说这样的点只有一个，这是因为这两个函数一个是增函数，另一个是减函数

如果蛛网收敛，则一般极限即为平衡点

按照实际情况，考虑取平衡点附近的某点，开始迭代，考察平衡点的稳定性

发现结论：该点处需求函数的斜率小于供应函数时，稳定；大于时，不稳定

在平衡点附近用直线近似曲线，就得到差分方程模型，由此得到的结论与上述结论一致

7、MSVC 初探

在 Windows 平台上，编写程序（尤其是 C/C++ 程序），需要针对 Windows 平台进行编译，那么就需要一个合适的编译器，Microsoft 提供了 MSVC 这样一个合适的编译器以供使用

如何使用 MSVC 呢？直接的方式就是使用 Visual Studio，目前看来，各种问题在网络上能找到的解答，多还是基于 Visual Studio 2019 的，或许说明 2019 仍是目前成熟的版本，而 2022 偏新，但实际操作显示，两个版本在基本功能上肯定还是兼容的，只是一些第三方库可能会对这个要求严格

那么既然是编译，就是把一堆源代码通过一堆命令生成出一堆中间文件，再通过一堆命令生成出最终文件，这些最终文件一般来说直接是 .exe，也可以是 .lib，.dll 之类的在 Windows 平台上的链接库

其实这一步道理都一样，在 Linux 上就是用 gcc 生成出 .o，.a 这种，其他平台也是一样

当项目达到一定规模后，手动进行这些操作将会变得不可取，于是就需要构建工具，与 MSVC 配套的构建工具是 CMake，正如在 gcc 中自带 make 以读取 Makefile 进行自动构建一样，CMake 读取 CMakeLists.txt 进行自动构建

写好了 CMakeLists.txt，现在直接用 CMake GUI 是非常方便的，最后构建到一个 build 文件夹里（这个文件夹是指定的，名字也不一定要叫 build）

然后在 Visual Studio 中打开项目，如果一切正常的话直接生成解决方案就可以了

这里开始要注意，有两个基本选项，如下图：

如果选择 Release，则生成（对了，VS 里生成等于编译）出来的东西是发行版，且文件名等信息与设置的一致，该是什么就是什么不会变

如果选择 Debug，则生成出来的东西是调试版，据观察得，文件名会在末尾加上一个d表示 Debug 模式（正如 Linux 中，想要正确链接一个链接库，实际上的链接库的文件名开头会加上lib一样，哦虽然这两者原理不同，但是至少…表观上来看很像…）
